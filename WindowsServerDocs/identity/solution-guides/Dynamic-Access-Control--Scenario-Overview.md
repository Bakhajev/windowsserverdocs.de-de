# <a name="dynamic-access-control-scenario-overview"></a>Dynamische Zugriffssteuerung: Szenarioübersicht

>Gilt für: Windows Server2016, Windows Server2012 R2, Windows Server 2012

In Windows Server2012 können Sie Daten governanceaktionen anwenden, auf Ihre Dateiserver zu steuern, wer auf Informationen zugreifen kann und zu überwachen, wer auf Informationen zugegriffen hat. Dynamische Zugriffssteuerung können Sie:  
  
-   Identifizieren von Daten mittels automatischer und manueller Klassifizierung von Dateien. Sie könnten beispielsweise Daten auf Dateiservern in der gesamten Organisation markieren.  
  
-   Steuern des Zugriffs auf Dateien durch Anwenden von Sicherheitsnetz-Richtlinien, die zentrale Zugriffsrichtlinien verwenden. Beispielsweise könnten Sie definieren, die Gesundheitsinformationen innerhalb der Organisation zugreifen können.  
  
-   Überwachen des Zugriffs auf Dateien mit zentralen Überwachungsrichtlinien für Kompatibilität und forensischen Analysen. Beispielsweise können Sie identifizieren, wer auf streng vertraulichen Informationen zugegriffen hat.  
  
-   Anwenden von Rights Management Services (RMS)-Schutz mithilfe der automatischen RMS-Verschlüsselung für vertrauliche Microsoft Office-Dokumente. Beispielsweise könnten Sie RMS so konfigurieren, alle Dokumente verschlüsselt werden, die Informationen von Health Insurance Portability and Accountability Act (HIPAA) enthalten.  
  
Die dynamische Zugriffssteuerung Featuresatz basiert auf Infrastrukturinvestitionen, die verwendeten weiter von Partnern und Line-of-Business-Anwendungen werden können, und die Features bieten interessante für Organisationen, die Active Directory verwenden. Diese Infrastruktur umfasst:  
  
-   Ein neues Autorisierungs- und Modul für Windows, die bedingte Ausdrücke und zentrale Richtlinien verarbeiten kann.  
  
-   Unterstützung des Kerberos-Authentifizierung für Benutzer- und Geräteansprüche.  
  
-   Verbesserungen der Dateiklassifizierungsinfrastruktur (FCI).  
  
-   RMS-Erweiterbarkeit unterstützen, damit Partner Lösungen bereitstellen können, die Nicht-Microsoft-Dateien zu verschlüsseln.
