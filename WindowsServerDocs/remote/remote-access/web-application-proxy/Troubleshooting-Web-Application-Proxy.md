# <a name="troubleshooting-web-application-proxy"></a>Problembehandlung: Webanwendungsproxy

>Gilt für: Windows Server 2016

**Dieser Inhalt ist für die lokale Version des Webanwendungsproxys relevant. Zum Aktivieren des sicheren Zugriffs auf lokale Anwendungen über die Cloud finden Sie unter den [Azure AD-Anwendungsproxy-Inhalt](https://azure.microsoft.com/documentation/articles/active-directory-application-proxy-get-started/).**  
  
Dieser Abschnitt enthält Verfahren zur Problembehandlung für Web Application Proxy, einschließlich Ereignis erläuterungen und Lösungen. Es gibt drei Stellen, in dem Fehler angezeigt werden:  
  
-   In der Web Application Proxy-Verwaltungskonsole  
  
    Jedes Ereignis-ID, die in der Administratorkonsole aufgeführt, die in der Windows-Ereignisanzeige und die entsprechenden Beschreibungen angezeigt werden kann, und unten Lösungen gefunden werden.  
  
    Öffnen der Ereignisanzeige, und suchen Sie nach Ereignissen, die im Zusammenhang mit der Web Application Proxy unter **Anwendungs- und Dienstprotokolle** > **Microsoft** > **Windows**  >  **Webanwendungsproxy** > **Admin**  
  
    ![](media/Troubleshooting-Web-Application-Proxy/WebApplicationProxyTroubleshooting.png)  
  
    Bei Bedarf, ausführliche Protokolle sind verfügbar, indem Analyse- und Debugprotokolle und Sitzungsprotokoll der Web Application Proxy, finden Sie in der Windows-Ereignisanzeige unter \ Microsoft \ Windows \ Web Application Proxy \ Admin.  
  
-   In PowerShell-Fehler  
  
    Ereignisse für auftretender Probleme bei der Konfiguration werden in PowerShell angezeigt.  
  
    Alle Fehler werden an der PowerShell-Benutzer, die mithilfe von standard-PowerShell-Fehler-aufforderungen angezeigt. Alle PowerShell-Befehle werden als Ereignisse protokolliert. Alle Ereignisse, die in PowerShell auftreten, finden Sie in der Windows-Ereignisanzeige mit der ID-Nummer 12016, und werden in den PowerShell-Abschnitt unten definiert.  
  
-   In der Best Practices Analyzer  
  
    Diese Ereignisse werden in beschrieben die [Best Practices Analyzer für den Webanwendungsproxy](assetId:///995f5cea-e334-4ce6-a14c-6a2d03c0c79a)
