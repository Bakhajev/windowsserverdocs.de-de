# <a name="always-on-vpn-features-and-functionalities"></a>Always On-VPN-Features und Funktionen

>Gilt für: WindowsServer (Halbjährlicher Kanal), WindowsServer 2016, Windows 10

- [**Vorherige:** Always On-VPN-Bereitstellung für Windows Server und Windows 10](always-on-vpn/deploy/always-on-vpn-deploy.md)
- [**nächster:** Erfahren Sie mehr über die Always On-VPN-Verbesserungen](../vpn/always-on-vpn/always-on-vpn-enhancements.md)

In diesem Thema erfahren Sie, über die Features und Funktionen von Always On-VPN.  In der folgende Tabelle ist keine vollständige Liste, aber es umfasst einige der am häufigsten verwendeten Features und Funktionen, die in Lösungen für den Remotezugriff verwendet. 

>[!TIP]
>Wenn Sie derzeit über DirectAccess verwenden, empfehlen wir, dass Sie, die Always On-VPN-Funktionalität sorgfältig durch, um zu bestimmen untersuchen, ob es sich bezieht, dass all Ihre remotezugriffsbedürfnisse erfüllt vor der Migration von DirectAccess zu Always On-VPN-bilden.

Jedes Element in diesem Abschnitt wird ein Anwendungsfall oder eine häufig verwendete RAS-Funktionen, die für die Always On-VPN-Funktionalität verbessert hat – entweder durch eine Erweiterung der Funktionalität oder Eliminierung von Duplikaten, der eine früheren Einschränkung.

Always On VPN ist in allen Windows-Versionen verfügbar, und die Plattformfeatures sind für Dritte über UWP VPN-Plug-In-Unterstützung verfügbar.<p><p>***Hinweis***:<br>Gerät-Tunnel kann nur auf die Domäne eingebundene Geräte, die Ausführung von Windows 10 Enterprise oder Bildungseinrichtungen Version 1709 oder höher konfiguriert werden. Darüber hinaus unterstützt Always On-VPN-OTP über MFA (nicht nativ unterstützt, nur auf Drittanbieter Plug-Ins unterstützt), über EAP-RADIUS-Integration.<p><p>Definieren Sie mithilfe von:<br>**VPNv2/ProfileName/NativeProfile/Authentication**                    |

Gruppenrichtlinien sind keine Abhängigkeit, um VPN-Profileinstellungen zu definieren, da Sie nicht während der Clientkonfiguration verwendet werden.
